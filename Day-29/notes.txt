Day 29 - Hooks

Topics Covered:
- useState: Manage local component state.
- useEffect: Handle side effects (API calls, subscriptions, timers).
- useContext: Consume context values directly.
- Custom Hooks: Reusable logic extracted into a separate function (e.g., useFetch, useCart).
- Rules of Hooks: Only call hooks at the top level, not inside loops/conditions.

How it applies to my E-Commerce App:
- Used useEffect for fetching data on component mount.
- Created a custom hook for API calls (e.g., useFetchData).
- Used useContext to access the cart context globally.

Key Learnings:
- useEffect dependency array controls when the effect runs (empty = run once, [dep] = run when dep changes).
- Cleanup function in useEffect (return () => {}) prevents memory leaks.
- Custom hooks must start with 'use' (e.g., useAxios).

Common custom hook pattern:
function useFetch(url) {
  const [data, setData] = useState(null);
  useEffect(() => { fetch(url).then(...); }, [url]);
  return data;
}
